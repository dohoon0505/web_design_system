# 05 · 쓰여지는 것도 시스템이다

사용자는 버튼과 문구를 분리하지 않습니다. **'저장'**과 **'저장하기'**는 다른 컴포넌트입니다. 여기서 정의하는 원칙은 모든 화면의 모든 문장에 적용됩니다.

---

## 7가지 라이팅 원칙

### PRINCIPLE 01 · 해요체로 일관되게

상황, 맥락, 화면 종류 불문 모든 문구는 해요체로 통일합니다. 합쇼체·반말·구어체 혼용은 브랜드 보이스를 깎아먹습니다.

| ❌ | ✅ |
| --- | --- |
| 저장되었습니다 | 저장했어요 |

### PRINCIPLE 02 · 능동형, 사용자가 주어

수동형은 주체를 모호하게 만듭니다. 사용자가 주체인 문장이 행동을 명확히 합니다.

| ❌ | ✅ |
| --- | --- |
| 이메일이 전송되었습니다 | 이메일을 보냈어요 |

### PRINCIPLE 03 · 부정 대신 긍정

"안 돼요"라고 말하지 말고, 무엇을 하면 되는지 알려줍니다. 사용자는 막힘이 아닌 길을 원합니다.

| ❌ | ✅ |
| --- | --- |
| 1,000원 이하는 송금할 수 없어요 | 1,000원부터 보낼 수 있어요 |

### PRINCIPLE 04 · 잡초 뽑기

"혹시", "앞으로", "방금", "일단", "정말" — 넣어도 빠져도 의미가 같으면 뺍니다. 모바일 사용자는 읽지 않고 스캔합니다.

| ❌ | ✅ |
| --- | --- |
| 혹시 앞으로 알림을 받으시려면 설정해주세요 | 알림을 받으려면 설정해주세요 |

### PRINCIPLE 05 · 버튼 라벨은 결과를 담는다

"확인"은 무엇을 확인하는지 모릅니다. 이 버튼을 누르면 무슨 일이 일어나는지 **동사**로 말합니다.

| ❌ | ✅ |
| --- | --- |
| 확인 · 완료 · OK | 결제하기 · 보내기 · 신청하기 |

### PRINCIPLE 06 · 에러는 막다른 길이 아니라 안내

왜 발생했는지 간결하게, 어떻게 해결할지 구체적으로. "다시 시도해주세요"는 아무것도 알려주지 않습니다.

| ❌ | ✅ |
| --- | --- |
| 오류가 발생했어요. 다시 시도해주세요 | 네트워크가 불안정해요. Wi-Fi를 확인해주세요 |

### PRINCIPLE 07 · 숫자는 정확하게, 단위는 일관되게

"잠시", "많은", "곧"은 라이팅이 아니라 회피입니다. 가능한 한 구체적인 숫자로. 단위 표기(₩, 원, %, MB, 분)는 한 제품에 한 방식으로 통일합니다.

| ❌ | ✅ |
| --- | --- |
| 잠시 후 다시 확인해주세요 | 3초 뒤에 다시 확인해주세요 |
| 용량이 부족해요 | 저장 공간이 500MB 남았어요 |
| 곧 만료되는 쿠폰이 있어요 | 23일 뒤에 만료되는 쿠폰이 있어요 |

## 컴포넌트별 라이팅 규칙

자주 등장하는 컴포넌트는 문구도 패턴화합니다. 고민의 여지를 줄여 일관성을 높이는 방법입니다.

| 컴포넌트 | 금지 | 권장 | 이유 |
| --- | --- | --- | --- |
| Dialog 왼쪽 버튼 | 취소 | 닫기 | '취소'는 지금까지 한 작업이 취소될까 봐 사용자가 망설입니다. |
| Dialog 오른쪽 버튼 | 확인 · OK · 네 | 삭제하기 · 나가기 · 저장하기 | 결과를 담는 동사여야 무엇이 일어나는지 예측됩니다. |
| Empty State 제목 | 데이터가 없습니다 | 아직 저장한 항목이 없어요 | 어떤 데이터인지 구체적으로. |
| Empty State 액션 | 추가 | 첫 항목 추가하기 | 비어 있다는 상황을 맥락으로 가져갑니다. |
| Toast 성공 | 성공하였습니다 | 저장했어요 | 일반적 단어('성공') 대신 구체적 행위. |
| Loading 상태 | Loading... · 잠시만요 | 불러오고 있어요 | 해요체 + 능동형. |
| Text Field 필수 표시 | (필수) | 빨간 별표 `*` | 단어는 공간을 먹고, 기호는 국제적으로 통용됩니다. |
| Form 입력 안내 | 올바른 이메일을 입력해주세요 | '@' 기호가 빠진 것 같아요 | 정확히 무엇이 문제인지 짚어줍니다. |

## 에러 메시지 템플릿

모든 에러는 세 가지 질문에 답해야 합니다:

> **무슨 일이 일어났나 · 왜 그런가 · 어떻게 할까**

### TEMPLATE A · 네트워크 오류

```
상황 + 원인 + 해결

연결에 실패했어요
네트워크가 불안정해요. Wi-Fi를 확인해주세요.
```

### TEMPLATE B · 유효성 검증 실패

```
어디서 + 무엇이 틀렸나

휴대폰 번호를 다시 확인해주세요
11자리 숫자를 모두 입력해주세요.
```

### TEMPLATE C · 시스템 점검

```
대상 + 시간 + 대안

하나은행 점검 중이에요
04월 21일 23시까지 송금이 어려워요. 다른 계좌를 써주세요.
```

## Navigating Error

에러 메시지의 역할은 "문제를 알림"이 아니라 **"다음 화면으로 넘어갈 수 있게 안내"**입니다. 메시지가 끝나도 사용자는 한 발자국 더 갈 수 있어야 합니다.
